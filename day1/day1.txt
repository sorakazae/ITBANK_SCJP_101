소스코드는 src파일 아래에 작성

class : 자바 프로그램(소스)의 기본단위     (c에서는 함수단위)
package : 자바class를 구분하는 단위 (폴더 생성)

실행(수행)시킬려면  main이 있어야 함

함수->메소드

public static void main(String[] args) {

(String[] args) 배열변수
자바에서는 꼭 써야함
args라는 변수명은 바꾸어도 됨
String args[]처럼 써도 상관 없다.

저장하면 자동 컴파일 됨(eclipse를 사용할때 해당)

api  :  프로그래밍 할 때 자주 쓰일 수 있는것들을 만들어 놓은것 (c언어의 구문)

자바는 참/거짓을 나타낼 수 있는 함수가 있다.(c는 함수가 따로 없고 0과1로 구분) 논리형

+연산자 : 1.덧샘연산    2. 문자열 결합(둘중 하나라도 문자열일 경우)
 문자열 결합에 많이 사용됨

print는 그냥 출력만 하는것 println은 출력후 행 추가

운영체제 = 소프트웨어 플랫폼

컴파일 : c언어 등 (실행속도가 빠르지만 사용이 한정되어 있다)


인터프리터 : 자바,html 등 
자바의 인터프리터(jre) => 실행 프로그램

자바는 컴파일&인터프리터 언어
자바의 파일은 바이트코드 파일이다
실행 하려면 실행 환경이 준비 되어 있어야함  =>  자바 인터프리터(jvm) 운영체제 역할

자바는 포인터가 없다(만세)

자바도 c와 마찬가지 기본적으로 int타입으로 변경후 연산

byte와 short형은 잘 사용하지 않는다.
주로 int와 long형을 사용

long(8)형보다 float(4)형이 더 넓은 범위를 저장할 수 있다.

c언어에서는 영어는 1byte 한글은 2byte 이였지만
자바에서는 영어도 2byte 한글도 2byte이다.

리터럴
boolean형 리터럴 true, false
True로 쓰면 변수로 인식

final(java언어) -> 상수 -> const(c언어) 

자바는 return타입을 무시하면 안됨(써야 한다) void라도 주어야 함

항의 개수가 적을수록 우선순위가 높다.

전달 = 대입  (연산이다.)
